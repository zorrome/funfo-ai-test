## Release summary
- Latest intent: 我需要一个餐厅预约系统。
系统的首页需要以单日视图的日历（カレンダー）形式展示，方便管理当天的预约情况。
首页需求
	•	首页只显示 1 天 的预约情况。
	•	页面主体采用 日历式布局：
	•	横轴是时间
	•	纵轴是餐桌
	•	用户可以在首页上 横向切换或选择日期，查看不同日期的预约情况。
	•	整体视觉上要像一个 便于管理的预约日历页面，而不是普通列表。

餐桌管理
	•	系统开始使用时，需要先创建餐桌。
	•	每个餐桌需要设置至少以下信息：
	•	餐桌名称或编号
	•	可容纳人数

预约管理
	•	用户可以在首页的日历（カレンダー）上直接点击某个时间段，为对应餐桌添加预约。
	•	创建预约时，需要填写：
	•	客户姓名
	•	电话号码
	•	预约人数
	•	预约时间
	•	备注信息（可选）

预约逻辑
	•	当预约人数 超过该餐桌可容纳人数 时，需要立即提示，不能直接预约成功。
	•	同一个时间段内，餐桌不能被重复预约，需避免冲突。

客户信息管理
	•	客户信息需要被保存下来，形成客户档案。
	•	下次输入客户电话号码时，系统可以 自动检索并补全已有客户信息。
	•	用户可以单独查看：
	•	客户信息列表
	•	某个客户的详细信息
	•	所有预约信息列表
	•	某一条预约的详细信息

目标
请把这个系统设计成一个适合餐饮门店使用的直观预约管理系统，重点突出：
	•	日历式首页管理体验
	•	单日预约一目了然
	•	客户信息可复用
	•	餐桌与人数限制清晰
	•	操作简单，适合门店现场使用
